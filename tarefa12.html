ATIVIDADES TEÓRICAS COM FOCO EM WINDOWS E LINUX E SEGURANÇA CIBERNÉTICA – TODA EXPLICAÇÕES DE FORMA PRÁTICA E GRÁFICA

(1) Definindo os Paradigmas: Código Aberto vs. Fechado

Um sistema operacional de código aberto é aquele que pode ser adquirido, editado e personalizado conforme o desejo do usuário. Já o sistema operacional de código fechado é fornecido por uma empresa, geralmente pago e sem possibilidade de edição do código.

Vantagens e Desvantagens

Código Aberto
Vantagens:

  • Gratuito
  • Mais leve
Desvantagens:
  • Menos seguro
  • Interface menos amigável

Código Fechado
Vantagens:

  • Maior suporte e segurança
  • Interface amigável
Desvantagens:
  • Geralmente pago
  • Pouca ou nenhuma personalização

Exemplos

Código Aberto: Ubuntu, Fedora, Tails
Código Fechado: Windows 10, Windows 11, macOS 15

(2) Análise Comparativa de Instalação

Durante a instalação, sistemas de código aberto como o Ubuntu oferecem mais opções de personalização, como o tipo de ambiente gráfico e a instalação de softwares adicionais. Já sistemas de código fechado como o Windows têm uma instalação mais guiada e simplificada, porém exigem a ativação por meio de licença paga. Em relação aos drivers, o Windows possui ampla compatibilidade automática, enquanto no Linux pode ser necessário instalar alguns manualmente.

(3) Debate: Modo Texto vs. Interface Gráfica

O modo texto (linha de comando) permite maior controle, automação e consumo reduzido de recursos. É ideal para servidores e usuários avançados. Já a interface gráfica (GUI) é intuitiva e amigável, indicada para tarefas cotidianas e novos usuários. Enquanto a GUI favorece a usabilidade, o modo texto é mais poderoso para operações complexas e manutenção de sistemas.

(4) Estruturas de Diretórios: Uma Análise Comparativa

Windows:

  • C:\Windows – Arquivos do sistema
  • C:\Program Files – Aplicativos instalados
  • C:\Users – Arquivos e configurações de usuários
Linux:
  • /bin – Programas essenciais
  • /etc – Arquivos de configuração
  • /home – Pastas dos usuários
Ambos os sistemas organizam arquivos de maneira lógica, mas o Linux tende a ser mais modular e voltado à administração por múltiplos usuários.

(5) Modelos de Usuários e Permissões

No Windows (NTFS), as permissões são atribuídas por meio de uma interface gráfica, com opções de controle total, leitura e gravação. No Linux, as permissões são representadas pelo sistema rwx (leitura, escrita, execução), com diferentes níveis para dono, grupo e outros. O Linux oferece maior granularidade e controle via terminal, sendo ideal para ambientes multiusuário.

(6) O Processo de Boot: Do Power-On ao Desktop

O processo de boot começa com o BIOS/UEFI, que verifica o hardware e procura o gerenciador de boot. No Linux, o GRUB é carregado, permitindo escolher entre sistemas. Em seguida, o kernel é iniciado, carregando módulos e processos essenciais até a interface gráfica. No Windows, o carregador de boot inicia o kernel, drivers e serviços antes de exibir a área de trabalho. A principal diferença está na abertura e personalização do processo em sistemas Linux.

(7) Redes: Compartilhamento, Mapeamento

a) Unidades e pastas em rede são recursos de armazenamento compartilhados em uma rede, permitindo o acesso remoto a arquivos e dados entre computadores. Servem para facilitar o trabalho colaborativo e centralizar informações.
b) Compartilhamento é tornar arquivos ou pastas acessíveis a outros usuários na rede. Acesso é a permissão dada para visualizar, modificar ou executar esses arquivos compartilhados.
c) Mapeamento de unidades de rede é o processo de atribuir uma letra de unidade (ex: Z:) a uma pasta de rede, tornando-a acessível como se fosse uma pasta local no computador do usuário.

(8) Fundamentos de Segurança Cibernética

a) Segurança Cibernética é o conjunto de práticas, tecnologias e processos usados para proteger sistemas, redes e dados contra ataques, danos ou acessos não autorizados.
b) 1. Vírus – Programas que infectam arquivos e se replicam.
2. Ransomware – Sequestra dados e exige resgate.
3. Phishing – Tentativa de enganar usuários para obter dados sensíveis.
4. Spyware – Espiona atividades do usuário.
5. Ataques DDoS – Sobrecarga de servidores para tirar serviços do ar.
c) Vulnerabilidades são falhas ou brechas em sistemas que podem ser exploradas por atacantes para comprometer a segurança.
d) Credenciais são os dados de autenticação de um usuário (como login e senha). Protegê-las é essencial para evitar acessos indevidos a contas e sistemas.
e) Engenharia Social é o uso de manipulação psicológica para obter informações confidenciais. Exemplos: fingir ser um técnico de suporte para obter senhas, ou enviar e-mails falsos para roubar dados.

(9) Métodos de Proteção e Prevenção

Para aumentar a segurança de sistemas e dados, usuários e administradores devem adotar medidas como: uso de senhas fortes, autenticação em dois fatores, backups regulares, atualização constante de softwares, e antivírus. Um firewall nativo do sistema monitora e controla o tráfego de rede, bloqueando conexões suspeitas e ajudando a prevenir invasões.

(10) Configuração do Sistema: Pontos Críticos

Áreas críticas de configuração incluem:
1. Gerenciamento de usuários e permissões – Controle de quem pode acessar e modificar arquivos.
2. Atualizações automáticas – Garantir que o sistema esteja sempre protegido contra falhas e ameaças.
3. Configuração de firewall e segurança de rede – Proteção contra acessos não autorizados.
4. Programas de inicialização – Otimização do desempenho ao controlar o que é carregado no boot.
5. Gerenciamento de energia e desempenho – Ajuste de uso de recursos conforme a necessidade.
